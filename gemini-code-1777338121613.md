# Especificación del Proyecto: Auto DayTrading Bot

## 1. Resumen del Proyecto
El objetivo es desarrollar un bot de daytrading 100% automatizado en Python que opere en el mercado de valores de EE. UU. utilizando la API de Alpaca. El bot tomará decisiones basadas en análisis técnico estricto (matemático) y, en una fase posterior, incorporará análisis de sentimiento de noticias financieras. 

## 2. Parámetros Financieros y Restricciones
* **Capital Inicial:** $450 dólares.
* **Tipo de Cuenta:** Cash Account (Cuenta de Efectivo).
* **Gestión PDT:** Al ser una cuenta de efectivo, no aplica la regla Pattern Day Trader (PDT).
* **Liquidación T+1:** El bot debe incluir lógica para leer el "saldo disponible real", entendiendo que el dinero de las ventas tarda un día hábil en liquidarse. Nunca debe intentar operar con fondos no liquidados.
* **Gestión de Riesgo Inquebrantable:** Toda operación de compra debe enviarse obligatoriamente como una "Bracket Order", incluyendo automáticamente un Stop-Loss y un Take-Profit.
* **Entorno de Trabajo:** Todo el desarrollo y las pruebas iniciales se harán estrictamente en el entorno de "Paper Trading" (Simulación) de Alpaca.

## 3. Stack Tecnológico Base
* **Lenguaje:** Python 3.10+
* **Conexión al Broker:** SDK oficial `alpaca-py` (para datos de mercado y ejecución de órdenes).
* **Manejo de Datos:** `pandas`.
* **Análisis Técnico:** `pandas-ta` o `TA-Lib`.
* **Seguridad de Credenciales:** `python-dotenv` (las API Keys y Secret Keys vivirán en un archivo `.env` local, nunca en el código fuente).
* **Despliegue Futuro:** El código debe estar estructurado para correr en un servidor Linux en la nube (VPS en GCP, AWS o DigitalOcean vía GitHub Student Pack) ejecutándose mediante `cron` o la librería `schedule`.

## 4. Arquitectura de Módulos (Fases de Desarrollo)
* **Módulo 1: Setup y Conexión:** Inicialización de `alpaca-py`, lectura de variables de entorno, y comprobación de estado de la cuenta.
* **Módulo 2: Motor de Datos (Data Feed):** Extracción de velas de precios históricas y en tiempo real usando la API gratuita de Alpaca.
* **Módulo 3: Motor de Análisis (Cerebro):** Cálculo de indicadores matemáticos (ej. Cruce de Medias Móviles o RSI) usando `pandas-ta`.
* **Módulo 4: Ejecución y Riesgo:** Lógica para validar saldo T+1 y envío de Bracket Orders.
* **Módulo 5: Sentimiento de Noticias (Experimental):** Extracción de titulares de Alpaca News y conexión con un LLM ligero para obtener un score (+1, 0, -1).

## 5. Instrucciones Estrictas para el Asistente de IA (Claude Code)
* Escribe código modular, con funciones claras y documentadas (Docstrings).
* Incluye manejo de errores (`try/except`) especialmente en las llamadas de red a la API de Alpaca.
* No utilices variables globales para los estados de las órdenes; mantén el estado consultando directamente la API de Alpaca.
* Jamás expongas variables de entorno en tus ejemplos de código generados.
